Name component: `Callout`  
Purpose: Shows a stylized call-out/alert box whose color and icon change according to its semantic “type.”  
Usage: `<Callout type="warning">Always back up your data.</Callout>`  
Props:  
• `children` – Content to be displayed inside the call-out box.  
• `type` – One of `"info" | "danger" | "warning" | "success"` (default is `"info"`).

Name component: `CodeTabs`  
Purpose: Displays a group of code blocks inside a tabbed interface, letting the user switch tabs, copy code, and expand/collapse long snippets. If they're just Terminal commands, give it a title="Terminal"
Usage:  
<CodeTabs defaultTab={0}>
  <CodeTab title="JavaScript">
```js
    console.log('Hello world');
```
  </CodeTab>
  <CodeTab title="Python">
```python
print("Hello world")`
```
  </CodeTab>
</CodeTabs>
Props:  
• `children` – One or more `<CodeTab>` elements.  
• `defaultTab` – Index of the tab that should be active on first render (number, default 0).  

Name component: `CodeTab`  
Purpose: Wraps an individual code block and supplies its title metadata for use by the parent `<CodeTabs>`.  
Usage: 
<CodeTab title="CSS">
```language
<code here>
```
</CodeTab>
Props:  
• `title` – Label text shown on the tab (string, required).  
• `children` – The code snippet or content to render when this tab is active.

Name component: `<Dos>`
Purpose: Presents a “Do” or “Don’t” guideline with the corresponding icon and the supplied explanatory text Atleast dos or donts must be true.

Usage:  
<Dos dos>Write concise, self-documenting code.</Dos>  
<Dos donts>Avoid using undocumented global variables.</Dos>

Props:  
• `dos` – boolean (default: false). When true, shows the green “Do” icon.  
• `donts` – boolean (default: false). When true, shows the red “Don’t” icon.  
• `children` – the guideline text or elements that appear next to the icon.

Name component: `<FileStructure>`
Purpose: Collapsible tree container that renders a hierarchy of files and folders and lets the user expand/collapse folders.
Usage:
<FileStructure>
  <File level={0} type="folder">src</File>
  <File level={1} type="file">index.js</File>

  <File level={0} type="folder">components</File>
  <File level={1} type="file">Button.jsx</File>
  <File level={1} type="file">Card.jsx</File>
</FileStructure>
Props:
• children – One or more `<File>` elements describing the structure.

Name component: `<File>`
Purpose: Single row inside `<FileStructure>` that can represent either a file or a folder (folders can be toggled open/closed).
Usage:
<File level={0} type="folder" focus>public</File>
<File level={1} type="file">favicon.svg</File>
Props:
• children – Label of the file/folder.  
• level – number (default: 0). Nesting depth that controls left padding.  
• type – "file" | "folder" (default: "file"). Decides icon and clickability.  
• focus – boolean (default: false). Highlights the row.

Name component: `<Steps>`  
Purpose: Container that automatically numbers each of its direct `<Step>` children and renders them in order. Works with all components. 
Usage:
<Steps>

  </Step>

  <Step title="Start the dev server">
    <p>Execute <code>npm run dev</code> and open <code>localhost:3000</code>.</p>
    <FileStructure>
      <File level={0} type="folder">src</File>
      <File level={1} type="file">index.js</File>
    </FileStructure>
  </Step>
</Steps>
Props:  
• `children` – One or more `<Step>` elements. No other props are required for `<Steps>` itself.

Name component: `<Step>`  
Purpose: Represents an individual, numbered instruction within a `<Steps>` list.  
Usage:
<Step title="Create a new file">
  <p>Name it <code>app.jsx</code> inside the <code>src</code> folder.</p>
</Step>
Props:  
• `title` – string. The heading displayed for the step.  
• `children` – JSX content that describes the action.  
• `stepNumber` – number (automatically injected by `<Steps>`; you normally omit it).
